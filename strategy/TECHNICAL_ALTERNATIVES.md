# 技術的代替手段・コスト削減策
> 最終更新: 2026-03-16
## コスト削減の技術施策
### 1. Routes API デバウンス（最大効果）
現状: WP追加のたびに即座に Routes API を呼び出し。WP20個で19回。
改善: デバウンス（3秒待ち）を入れるだけで19回→5〜6回に削減。
さらに「ルート計算」ボタンを手動にすれば1回。
効果: 月額 Routes API コスト 970円→100〜200円（1ユーザー）。
### 2. サムネイルの遅延生成
現状: 保存のたびに Static Maps API を呼び出し。
改善: TOP画面で実際に表示されるときに初めて生成。編集途中の無駄な生成を排除。
### 3. Nearby Search の代替
Nearby Search は $32/1,000 で最高額。
代替: ユーザーが Autocomplete（$2.83/1,000）で「ガソリンスタンド」と手入力。
UXは劣るがコストは1/11。Pro プラン限定にする方法もあり。
### 4. Places Photos のキャッシュ強化
Photos は Enterprise SKU で無料枠が1,000回/月と少ない。
写真URLの有効期限管理を強化し、不要な再取得を減らす。
### 5. KMLインポートによるルート作成（API呼び出しゼロ）
ユーザーが Google マイマップでルートを作成 → KMLエクスポート → FlexRoute にインポート。
ルート作成段階では API コストゼロ。ナビ開始時のみ Routes API を呼ぶ。
## 代替ルーティングエンジン
### OSRM (Open Source Routing Machine)
- ライセンス: BSD-2-Clause（無料）
- データ: OpenStreetMap
- 自前サーバーに構築可能
- 日本語案内文: 英語ベース（日本語化は追加開発が必要）
- 道路種別: OSM の `ref` タグ（国道254号等）から判定可能
- ルート計算速度: 高速（ミリ秒単位）
### Valhalla
- ライセンス: MIT（無料）
- データ: OpenStreetMap
- OSRM より高機能（交通規制、ターンバイターン案内が充実）
- Mapbox が内部で使用
- 日本語案内: テンプレートベースで多言語対応可能
### Mapbox Directions API
- 従量課金だが Google より安い
- 日本語案内対応
- 月100,000リクエストまで無料
### OSM の日本データの品質
- 主要国道・県道: 概ねカバー済み。`ref` タグで道路番号を取得可能
- 市町村道・私道: ゼンリンに比べて不完全
- FlexRoute のセールスポイント（道路種別色分け・日本語案内文）への影響:
  - 道路種別色分け: OSM タグから実現可能だが、Google Routes API の案内文ベースより精度が落ちる場面あり
  - 日本語案内文: 追加開発が必要。「国道254号を左折」のような自然な案内は Google の品質に及ばない
## Phase 別のコスト戦略（提案）
| フェーズ | 方式 | 月額コスト |
|---|---|---|
| Phase 1（現在〜少人数） | FlexRoute の1キーで運営、無料枠内 | ¥0 |
| Phase 2 初期 | Routes API デバウンス + 手動化 | 大幅削減 |
| Phase 2 サブスク開始 | Rider/Pro 課金で API コスト回収 | 黒字化目標 |
| Phase 2 拡大期 | OSRM/Valhalla 自前サーバーに段階移行 | Google API 依存脱却 |
## 未解決の根本課題
上記の削減策を全て適用しても、Google Maps JavaScript API（地図表示自体）の従量課金は残る。
地図表示を完全に無料にするには、Leaflet + OpenStreetMap タイル等への移行が必要だが、
Google Maps の UX（Street View、Place 情報、高品質タイル）を犠牲にすることになる。
**APIコストを根本的にゼロにする方法は現時点で存在しない。**
削減と収益化の組み合わせで採算を取る戦略が現実的。
