## 料金体系の変更（2025年3月1日〜）
旧制度の月間 $200 クレジットが廃止され、SKU ごとの無料利用枠に変更された。
| カテゴリ | 無料枠/SKU/月 |
|---|---|
| Essentials | 10,000回 |
| Pro | 5,000回 |
| Enterprise | 1,000回 |
無料枠は**請求先アカウント（Billing Account）単位**で適用。
同一請求先にリンクされた全プロジェクトの利用が合算される。
全 SKU の無料枠を合計すると最大 $3,250/月相当の無料利用が可能（旧制度の $200 から大幅拡大）。

## 1ユーザーの月間利用パターン（想定）
### 前提条件
- ルート新規作成: 30本/月（1本/日、WP20個、ほぼ全て Place 検索）
- 既存ルート再編集: 10回/月
- ナビゲーション: 4回/月（週1回往復600km）
- TOP画面表示: 150回/月（5回/日）
- Place アイコンタップ: 300回/月（10回/日）
- リルート: 10回/月（ナビ中の逸脱）
- 保有データ: ルート100本、場所500件

> **Roads API（不採用、参考試算）**: リアルタイムスナップ（1点/3秒 × 20時間/月）= 24,000回/月（無料枠の480%）。10点バッチ（10秒間隔）= 2,400回/月（48%）だが UX が劣る。Phase 1 ではポリラインの最近点スナップを採用（D-027）。

**結論: 1ユーザーが個人の無料枠を使い切ることは現実的にありえない。**
最も消費率が高い Photos でも20%。ヘビーユーザーが通常の3倍使っても最大60%。

### 500人付近がピークになる理由
Nearby Search（$32/1,000、最高額SKU）が100人でちょうど無料枠5,000回の境界にあり、
101人目から突然課金が開始。ボリューム割引は10万コール〜で効き始めるため、
この規模ではまだ恩恵がない。

## Routes API が高い根本原因
現在の実装（useRouteCalculation）は WP 追加のたびに Routes API を即座に呼び出す。
WP20個のルートで19回。デバウンスも手動ボタンもない。
