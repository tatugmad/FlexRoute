# 収益化戦略
> 最終更新: 2026-03-16
## フリーミアムモデル案
| プラン | 月額 | 内容 |
|---|---|---|
| Free | ¥0 | ルート3本まで、ナビなし |
| Rider | ¥980 | ルート無制限、ナビ、走行記録 |
| Rider Pro | ¥1,980 | 全機能 + クモの巣走破地図 + KML + 複数デバイス同期 |
### 損益分岐（APIコスト削減後 ≈ 500円/ユーザーを想定）
| プラン | 人数 | APIコスト | 粗利 |
|---|---|---|---|
| Free | - | ~50円（制限あり） | -50円 |
| Rider | - | ~500円 | +480円 |
| Pro | - | ~600円 | +1,380円 |
### ユーザー構成 70:20:10 仮定（1,000ユーザー時）
| プラン | 人数 | 月間売上 | 月間APIコスト | 粗利 |
|---|---|---|---|---|
| Free | 700 | ¥0 | ¥35,000 | -¥35,000 |
| Rider | 200 | ¥196,000 | ¥100,000 | +¥96,000 |
| Pro | 100 | ¥198,000 | ¥60,000 | +¥138,000 |
| **合計** | **1,000** | **¥394,000** | **¥195,000** | **+¥199,000** |
サーバー費用（月3〜5万円）を差し引いても月15万円程度の利益。
### 前提条件
- 上記の「500円/ユーザー」は Routes API デバウンス + 手動化などの技術的コスト削減を適用した後の想定
- 削減なしの現状では1人あたり約1,500〜1,800円で、Rider プランでは赤字
- **収益化にはコスト削減が必須条件**
## 未解決の課題
- APIコスト削減が収益化の前提条件だが、最適な削減方法が未確定
- Free プランの制限（ルート3本、ナビなし）で API コールがほぼ発生しない設計が重要
- Nearby Search（$32/1,000）を Pro プラン限定にすることでコスト集中を回避可能
